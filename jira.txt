To check **Service Accounts (SA)** in **GKE** using `kubectl`, you can use the following commands.

---

## 1️⃣ List all Service Accounts in a namespace

```bash
kubectl get sa -n <namespace>
```

Example:

```bash
kubectl get sa -n default
```

Output example:

```bash
NAME      SECRETS   AGE
default   1         25d
app-sa    1         10d
```

---

## 2️⃣ List Service Accounts in all namespaces

```bash
kubectl get sa --all-namespaces
```

or

```bash
kubectl get serviceaccounts -A
```

---

## 3️⃣ Describe a specific Service Account

```bash
kubectl describe sa <sa-name> -n <namespace>
```

Example:

```bash
kubectl describe sa app-sa -n default
```

This shows:

* Labels
* Annotations
* Tokens
* Image pull secrets

---

## 4️⃣ Check which Service Account a Pod is using

Very important in GKE 👇

```bash
kubectl get pod <pod-name> -n <namespace> -o jsonpath='{.spec.serviceAccountName}'
```

Example:

```bash
kubectl get pod my-app-pod -n default -o jsonpath='{.spec.serviceAccountName}'
```

If nothing is specified, it uses:

```
default
```

---

## 5️⃣ If using Workload Identity (GKE → GCP SA mapping)

In GKE with **Workload Identity**, Kubernetes SA is annotated with a GCP SA.

Check annotation:

```bash
kubectl get sa <sa-name> -n <namespace> -o yaml
```

Look for:

```yaml
annotations:
  iam.gke.io/gcp-service-account: my-gcp-sa@project-id.iam.gserviceaccount.com
```

That tells you which **GCP Service Account** is mapped.

---

## 6️⃣ Quick way to see SA for all pods in a namespace

```bash
kubectl get pods -n <namespace> -o custom-columns=NAME:.metadata.name,SA:.spec.serviceAccountName
```

Example:

```bash
kubectl get pods -n default -o custom-columns=NAME:.metadata.name,SA:.spec.serviceAccountName
```

---

# 🎯 Summary

| What you want                   | Command                                                          |
| ------------------------------- | ---------------------------------------------------------------- |
| List SAs                        | `kubectl get sa -n <ns>`                                         |
| Describe SA                     | `kubectl describe sa <name> -n <ns>`                             |
| Pod's SA                        | `kubectl get pod <pod> -o jsonpath='{.spec.serviceAccountName}'` |
| Check Workload Identity mapping | `kubectl get sa <name> -o yaml`                                  |
