In Kubernetes (like in your **GKE cluster** for the ID Card System project), you can create a namespace using simple `kubectl` commands.

---

## ✅ 1️⃣ Create Namespace (Direct Command)

```bash
kubectl create namespace <namespace-name>
```

### Example:

```bash
kubectl create namespace mics-dev
```

### Verify:

```bash
kubectl get namespaces
```

---

## ✅ 2️⃣ Create Namespace Using YAML (Recommended in Enterprise Projects)

In production projects (like yours with **Git + GKE + CI/CD**), YAML-based creation is preferred.

### Step 1: Create a file `namespace.yaml`

```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: mics-dev
  labels:
    environment: dev
```

### Step 2: Apply it

```bash
kubectl apply -f namespace.yaml
```

---

## ✅ 3️⃣ Check Namespace Details

```bash
kubectl describe namespace mics-dev
```

---

## ✅ 4️⃣ Deploy Resources into That Namespace

Example:

```bash
kubectl apply -f deployment.yaml -n mics-dev
```

Or set default namespace for your session:

```bash
kubectl config set-context --current --namespace=mics-dev
```
