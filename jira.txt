Below is the **complete CI piece you should add to your existing workflow** (`mics-ui-components-ci.yaml`) to:

1. Run **after `docker-push`**
2. Generate the **same Docker tag used during push**
3. Update **`k8s-manifests/deployment.yaml`**
4. Commit the change back to the **same branch that triggered the workflow**

I’ll also explain **exactly where to insert it**.

---

# 1️⃣ Add Permission (Top of Workflow)

At the **top of your workflow file**, add this if not present:

```yaml
permissions:
  contents: write
```

This allows the workflow to **commit changes back to the repo**.

---

# 2️⃣ Add This Job AFTER `docker-push`

In your current pipeline you have:

```
notify-qa-deployment-approval
        ↓
docker-build
        ↓
docker-push
```

Add **this job immediately after `docker-push`**.

---

# 3️⃣ Complete CI Job to Add

```yaml
update-deployment-manifest:
  name: Update Kubernetes Deployment Image
  needs: docker-push
  runs-on: self-hosted

  steps:

    - name: Checkout Repository
      uses: actions/checkout@v4
      with:
        ref: ${{ github.ref_name }}

    - name: Print Branch
      run: |
        echo "Running on branch: $GITHUB_REF_NAME"

    - name: Install yq
      run: |
        sudo curl -L https://github.com/mikefarah/yq/releases/latest/download/yq_linux_amd64 -o /usr/local/bin/yq
        sudo chmod +x /usr/local/bin/yq
        yq --version

    - name: Generate Docker Tag
      id: tag
      run: |
        TAG="${{ github.event.inputs.deployment-environment }}-${{ github.sha }}-${{ github.run_number }}"
        echo "tag=$TAG" >> $GITHUB_OUTPUT
        echo "Generated Docker tag: $TAG"

    - name: Update deployment.yaml
      run: |

        FILE="k8s-manifests/deployment.yaml"

        TAG="${{ steps.tag.outputs.tag }}"

        IMAGE_REPO="cvsh.jfrog.io/cvsdigital-docker/ghemu/cvs-health-source-code/mics-ui-components/mics-ui-components"

        NEW_IMAGE="$IMAGE_REPO:$TAG"

        echo "New image will be: $NEW_IMAGE"

        export NEW_IMAGE

        # Print old image
        echo "Old image:"
        yq e '.spec.template.spec.containers[] | select(.name=="mics-ui-components").image' $FILE

        # Update YAML safely
        yq e '(.spec.template.spec.containers[] | select(.name=="mics-ui-components").image) = strenv(NEW_IMAGE)' -i $FILE

        echo "Updated image:"
        yq e '.spec.template.spec.containers[] | select(.name=="mics-ui-components").image' $FILE

    - name: Commit and Push Changes
      run: |

        git config user.name "github-actions"
        git config user.email "github-actions@github.com"

        if [[ -n $(git status --porcelain) ]]; then
          git add k8s-manifests/deployment.yaml
          git commit -m "Update deployment image to ${{ steps.tag.outputs.tag }}"
          git push origin $GITHUB_REF_NAME
        else
          echo "No changes detected"
        fi
```

---

# 4️⃣ What This Job Does (Step-by-Step)

### Step 1 — Checkout the Same Branch

```yaml
ref: ${{ github.ref_name }}
```

This ensures:

| Workflow Trigger | File Updated In     |
| ---------------- | ------------------- |
| main             | main                |
| dev              | dev                 |
| feature branch   | same feature branch |

---

### Step 2 — Install `yq`

Used to **safely modify YAML**.

---

### Step 3 — Generate Docker Tag

Same format used in your existing pipeline:

```bash
ENV-SHA-RUNNUMBER
```

Example:

```
dev-5f60faa02b4a54499c6311502d24c5fd8dd021e6-142
```

---

### Step 4 — Update YAML Image Field

Your YAML path:

```
spec
 └ template
    └ spec
       └ containers
          └ name: mics-ui-components
             image: ...
```

The safe update query:

```bash
(.spec.template.spec.containers[] | select(.name=="mics-ui-components").image)
```

This ensures it **updates the correct container even if order changes**.

---

### Step 5 — Commit Changes

Only commits if file actually changed.

```
git status --porcelain
```

Then pushes to:

```
origin $GITHUB_REF_NAME
```

```
        ↓

        ↓

```

This is **proper GitOps deployment behavior**.
