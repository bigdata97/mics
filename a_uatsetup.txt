---

## ✅ Full YAML Template

```yaml
name: Enterprise CI-CD Pipeline

on:
  workflow_dispatch:
    inputs:
      environment:
        description: "Select deployment environment"
        required: true
        type: choice
        options:
          - dev
          - qa
          - uat
          - prod

env:
  IMAGE_NAME: your-docker-image

jobs:

  # =========================
  # 1️⃣ CI JOB (ALL BRANCHES)
  # =========================
  build:
    runs-on: ubuntu-latest

    outputs:
      image_tag: ${{ steps.set_tag.outputs.tag }}

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set Image Tag
        id: set_tag
        run: |
          if [[ "${{ github.ref_name }}" == "develop" || "${{ github.ref_name }}" == "main" || "${{ github.ref_name }}" == release/* ]]; then
            echo "tag=${GITHUB_SHA}" >> $GITHUB_OUTPUT
          else
            echo "tag=${GITHUB_SHA}-SNAPSHOT" >> $GITHUB_OUTPUT
          fi

      - name: Docker Build
        run: |
          docker build -t $IMAGE_NAME:${{ steps.set_tag.outputs.tag }} .

  # =========================
  # 2️⃣ DEPLOY JOB (RESTRICTED)
  # =========================
  deploy:
    needs: build
    runs-on: ubuntu-latest

    # 🚨 Block feature branches entirely
    if: |
      github.ref_name == 'develop' ||
      github.ref_name == 'main' ||
      startsWith(github.ref_name, 'release/')

    steps:

      - name: Validate Branch-Environment Mapping
        run: |
          BRANCH="${{ github.ref_name }}"
          ENV="${{ inputs.environment }}"

          if [[ "$BRANCH" == "develop" && "$ENV" != "dev" ]]; then
            echo "❌ Develop branch can deploy only to DEV"
            exit 1
          fi

          if [[ "$BRANCH" == "main" && "$ENV" != "prod" ]]; then
            echo "❌ Main branch can deploy only to PROD"
            exit 1
          fi

          if [[ "$BRANCH" == release/* && "$ENV" != "qa" && "$ENV" != "uat" ]]; then
            echo "❌ Release branch can deploy only to QA or UAT"
            exit 1
          fi

          echo "✅ Branch and environment validation passed"

      - name: Docker Push
        run: |
          docker push $IMAGE_NAME:${{ needs.build.outputs.image_tag }}

      - name: Update Deployment Manifest
        run: |
          echo "Updating manifest for ${{ inputs.environment }}"
          # your sed / yq logic here

      - name: Create PR
        run: |
          echo "Creating PR..."
          # gh pr create logic
```

# 🧠 Why This Is Correct

### ✔ Feature branches

* Build runs
* No push
* No deployment
* No manifest update
* No PR

### ✔ develop branch

* Can deploy ONLY to dev
* Cannot deploy to qa or prod

### ✔ release branch

* Can deploy only to qa or uat

### ✔ main branch

* Can deploy only to prod

# 🔒 Why Validation Must Be Before Docker Push

You were right here.

```

```

Even if someone manually pulls image:

* Snapshot images clearly marked
* No accidental deployment
