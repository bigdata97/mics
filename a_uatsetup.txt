## **Description**

This story covers the production deployment of the `mics-ui-components` repository along with the dependent `logos` and `login` repositories to complete the end-to-end PROD release flow.

As part of this deployment, all application dependencies required for successful PROD access and functionality need to be configured. This includes setting up the `MICS-PROD` client in Ping, enabling Zero Trust access, updating the refresh interval configuration to 60 seconds, and ensuring the required data is created or migrated in MongoDB to support the logo service and associated UI functionality.

The objective of this story is to make the PROD environment fully functional from both infrastructure and application access perspectives, ensuring login, branding, and UI rendering work seamlessly after deployment.

---

## **Acceptance Criteria**

The `mics-ui-components` repository is deployed successfully to the PROD environment.

Dependent repositories `logos` and `login` are also deployed successfully as part of the same release cycle.

`MICS-PROD` client is created and configured in Ping with the required redirect URIs, client settings, and access policies.

Zero Trust access is enabled and validated for PROD users.

Application refresh interval is updated to 60 seconds and verified post deployment.

Required MongoDB data is created or migrated successfully to support the logo service and UI rendering.

Login flow, logo rendering, and UI component loading are validated successfully in PROD.

Smoke testing is completed and stakeholder confirmation is received.

---

## **Definition of Done**

All three repositories are successfully deployed to PROD.

Ping client configuration is completed and validated.

Zero Trust and authentication flows are working as expected.

MongoDB data setup is completed and verified.

Post-deployment smoke testing is successful with no major issues.

Deployment evidence, validation screenshots, and implementation notes are added to the story.

The PROD environment is ready for business / user validation.

---

## **Change Implemented**

Deployed `mics-ui-components`, `logos`, and `login` repositories to PROD.

Configured `MICS-PROD` client in Ping, enabled Zero Trust access, updated refresh interval to 60 seconds, and completed MongoDB data setup required for logo service functionality.

Validated login flow, UI rendering, and logo service response successfully in PROD.
