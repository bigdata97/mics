Use this complete approach in your GitHub Actions job:

* create a temporary **user-level `.npmrc`**
* run `npm ci`
* delete the temporary `.npmrc` afterward

This is the cleanest fix for your case.

## Why this is the right solution

Your repo `.npmrc` should keep only:

* registry URL
* non-secret npm settings

Your workflow should handle:

* token injection
* cleanup after install

That avoids storing secrets in the repo.

---

# 1) Keep repo `.npmrc` like this

In the repo root `.npmrc`, keep it simple:

```ini
engine-strict=true
package-lock=true
always-auth=true
registry=https://cvsh.jfrog.io/artifactory/api/npm/cvs-aetnahealth-npm/
```

Do **not** keep token lines there.

---

# 2) Update workflow with create + delete `.npmrc`

In your `test-and-sonar` job, use this full block:

```yaml
test-and-sonar:
  needs: [generate-token]
  runs-on: self-hosted
  name: Run Tests and Internal Sonar Scan
  steps:
    - uses: actions/checkout@v4
      with:
        fetch-depth: 0

    - name: Setup Node.js
      uses: actions/setup-node@v4
      with:
        node-version: '22.21.1'

    - name: Create temporary npm auth file
      shell: bash
      run: |
        cat > ~/.npmrc <<EOF
        engine-strict=true
        package-lock=true
        always-auth=true
        registry=https://cvsh.jfrog.io/artifactory/api/npm/cvs-aetnahealth-npm/
        //cvsh.jfrog.io/artifactory/api/npm/cvs-aetnahealth-npm/:_authToken=${JFROG_READER_TOKEN}
        EOF
      env:
        JFROG_READER_TOKEN: ${{ secrets.JFROG_READER_TOKEN }}

    - name: Install Dependencies
      run: |
        rm -rf node_modules
        npm ci

    - name: Delete temporary npm auth file
      if: always()
      shell: bash
      run: |
        rm -f ~/.npmrc

    - name: Run Tests with Coverage
      run: npm run test:coverage
      continue-on-error: true

    - name: CVS Internal Sonar Scan
      uses: cvs-health-source-code/gha_workflow_actions/actions/sonar_scan@beta
      with:
        PROJECT_KEY: ${{ github.repository_owner }}_${{ github.event.repository.name }}
        PROJECT_NAME: mics-ui-components
        SONAR_TOKEN: ${{ secrets.SONAR_TOKEN }}
        SONAR_ANALYSIS_TOKEN: ${{ secrets.SONAR_ANALYSIS_TOKEN }}
        INPUT_ARGS: >
          -Dsonar.sources=src
          -Dsonar.tests=src
          -Dsonar.javascript.lcov.reportPaths=coverage/lcov.info
          -Dsonar.test.inclusions=**/*.spec.js,**/*.test.js,**/*.spec.jsx,**/*.test.jsx
          -Dsonar.coverage.exclusions=**/*.spec.js,**/*.test.js,**/*.spec.jsx,**/*.test.jsx,**/setupTests.js,**/main.jsx,**/__mocks__/**
```

# 4) Safer version with backup and restore

If your self-hosted runner may already have a `~/.npmrc`, use this instead so you don’t overwrite it permanently.

```yaml
- name: Backup existing npmrc if present
  shell: bash
  run: |
    if [ -f ~/.npmrc ]; then
      mv ~/.npmrc ~/.npmrc.backup

- name: Create temporary npm auth file

    //cvsh.jfrog.io/artifactory/api/npm/cvs-aetnahealth-npm/:_authToken=${JFROG_READER_TOKEN}
    EOF
  env:
    JFROG_READER_TOKEN: ${{ secrets.JFROG_READER_TOKEN }}

- name: Install Dependencies

- name: Cleanup and restore npmrc
```
