Branch-Aware CI Setup Steps for Java Backend Repositories

These steps are applicable for Java-based backend service repositories in the MICS project.

1. Update the CI workflow file

Go to the repository for your backend service.

Navigate to:

.github/workflows/ci.yaml

Open the reference CI workflow from:

gha_workflow_actions/.github/workflows/maven_docker.yaml

Copy the complete content from that file and paste it into your service repo’s:

.github/workflows/ci.yaml

Commit this change to the appropriate branch.

2. Add required repository variables

In your service repository, go to:

Settings → Secrets and variables → Actions → Variables

Click New repository variable and create the following variables.

Variable 1
Name: ALLOWED_BRANCHES
Value: develop,release/*,main
Variable 2
Name: APP_NAME
Value: <repo-name>

Example:

Name: APP_NAME
Value: mics-logo-service
Variable 3
Name: WEBHOOK_MICS_RELEASE_APPROVALS_C...
Value: <use the approved Teams/webhook URL shared for MICS release approvals>

Use the exact variable name and webhook value provided in the shared setup note. These names and values are case-sensitive.

3. Update repository custom property

Go to:

Settings → Custom properties

Scroll down and find:

ruleset-compliance

Change the value from:

none

to:

standard

Save the change.

This is required for the branch protection / SOD gating used by the reusable CI workflow.

4. Ensure develop branch is available

If the repository already has a develop branch, make sure it is up to date with main.

If the repository does not have a develop branch:

Create a new develop branch from main.

Then create a PR:

From: main
To: develop

Merge the PR so that develop has the latest code from main.

5. Run the branch-aware CI workflow

After the CI file, repository variables, custom property, and branch setup are complete:

Go to:

Actions

Select the workflow:

Branch Aware CI Workflow

Click:

Run workflow

Use the following selections:

Use workflow from: develop
Select the environment to deploy to: dev

Keep the remaining options unchanged unless specifically required.

Then click:

Run workflow
Notes

These steps are for Java-based backend services only. Python-based services may require slightly different workflow configuration.
