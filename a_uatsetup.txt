## **Description**

As a Cloud Engineer, I want to update the DEV environment DNS to point to the new application endpoint and retire the old DNS entry so that all traffic is routed through the latest infrastructure setup and the legacy configuration can be safely sunset.

This story covers the DNS cutover activity for the DEV environment as part of the ongoing platform modernization and environment standardization effort. The work includes identifying the currently active DNS record, updating it to point to the new ingress / load balancer endpoint, validating application accessibility through the updated DEV URL, and decommissioning the old DNS record after successful verification.

The objective is to ensure a seamless transition with no disruption to existing development workflows and to clean up outdated DNS configurations that are no longer in use.

---

## **Acceptance Criteria**

The DEV DNS record is updated successfully and resolves to the new target endpoint.

The application is accessible through the updated DEV URL and all primary routes are reachable.

SSL / HTTPS validation is successful and the certificate is mapped correctly to the new DNS.

The legacy DNS entry is identified and retired after successful validation of the new endpoint.

Smoke testing is completed with confirmation from the application team that the environment is functioning as expected.

A rollback approach is documented in case any issues are identified during or after the cutover.

---

## **Definition of Done**

DNS cutover for the DEV environment is completed successfully.

Application accessibility and routing validation are completed.

Old DNS record has been removed or marked for decommission as per platform standards.

Relevant validation evidence such as browser access confirmation, DNS lookup results, or screenshots is attached to the story.

Implementation details and any follow-up actions are documented in the JIRA comments.

Stakeholder confirmation is received and the story is ready to move to Done.

---

## **Change Implemented**

Updated the DEV DNS entry to point to the new ingress endpoint and validated successful application access through the updated URL.

Confirmed DNS resolution, SSL mapping, and route accessibility. Legacy DNS entry has been identified for sunset and cleanup completed post-validation.
