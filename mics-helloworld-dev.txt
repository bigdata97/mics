Run this with your **prod project id** and **infra SA email**:

```bash
PROD_PROJECT_ID="hcb-prod-mics"
INFRA_SA="infra-admin@hcb-prod-mics.iam.gserviceaccount.com"

gcloud projects get-iam-policy "$PROD_PROJECT_ID" \
  --flatten="bindings[].members" \
  --filter="bindings.members:serviceAccount:$INFRA_SA" \
  --format="table(bindings.role)"
```

If your infra SA email is different, replace it.

To see full details including IAM conditions:

```bash
gcloud projects get-iam-policy "$PROD_PROJECT_ID" \
  --flatten="bindings[].members" \
  --filter="bindings.members:serviceAccount:$INFRA_SA" \
  --format="yaml(bindings.role,bindings.members,bindings.condition)"
```

Also check who can impersonate that infra SA:

```bash
gcloud iam service-accounts get-iam-policy "$INFRA_SA" \
  --project="$PROD_PROJECT_ID"
```

For folder/org inherited roles, project policy alone may not show everything. Use this too:

```bash
gcloud asset search-all-iam-policies \
  --scope="projects/$PROD_PROJECT_ID" \
  --query="policy:$INFRA_SA" \
  --format="table(resource,policy.bindings.role)"
```
