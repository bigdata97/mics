# 3️⃣ User Story Description

Here is a **professional description you can paste in JIRA**.

---

### User Story

Currently, after the CI pipeline builds and pushes a Docker image to **JFrog Artifactory**, the generated Docker image tag must be manually copied and updated in the Kubernetes `deployment.yaml` file located under the `k8-manifests` directory.

Since our deployments follow a **GitOps model using ArgoCD**, the Kubernetes manifests repository acts as the source of truth. Any change in the manifests triggers ArgoCD to synchronize and deploy the new version to the cluster.

However, the current manual process introduces several challenges:

* Manual effort after every CI build
* Risk of human error when copying image tags
* Delayed deployments due to manual intervention
* Inconsistent deployment practices

To improve the CI/CD workflow, we propose implementing an automation mechanism where the CI pipeline automatically updates the Docker image tag in the Kubernetes manifest files and commits the change to the repository. This will allow ArgoCD to automatically detect the update and deploy the latest image.

This enhancement will ensure a **fully automated GitOps-based deployment workflow**.

---

# 4️⃣ Acceptance Criteria

### AC1

When the **CI workflow completes successfully**, the generated Docker image tag should be automatically captured.

### AC2

The CI pipeline should update the Docker image tag in the appropriate Kubernetes `deployment.yaml` file inside the `k8-manifests` directory.

Example:

Before:

```
image: jfrog.company.com/my-app:old-tag
```

After:

```
image: jfrog.company.com/my-app:new-ci-tag
```

---

### AC3

The pipeline should automatically:

* commit the change
* push the update to the target branch.

---

### AC4

Once the manifest repository is updated, **ArgoCD should automatically detect the change and sync the deployment**.

---

### AC5

The automation should support **environment-specific updates**, such as:

* dev
* qa
* uat
* prod

---

### AC6

The pipeline should include validation to ensure the image tag update was successful before committing changes.

---

# 5️⃣ Technical Approach (Implementation Guidance)

This is helpful for whoever implements the story.

### Step 1

Capture image tag from CI build.

Example:

```
IMAGE_TAG=${{ github.sha }}
```

---

### Step 2

Update deployment file automatically.

Example:

```
sed -i "s|image: .*|image: jfrog.company.com/my-app:${IMAGE_TAG}|g" k8-manifests/deployment.yaml
```

---

### Step 3

Commit changes

Example:

```
git config user.name "github-actions"
git config user.email "github-actions@company.com"

git add k8-manifests/deployment.yaml
git commit -m "Update image tag to ${IMAGE_TAG}"
git push origin main
```

---

### Step 4

ArgoCD auto-sync deploys the change.
